# 编程概念

# 对象

  • 状态 -> 属性
  • 表现 -> 方法

观察现实世界的物体，考虑他们有什么属性和可能被操作导致属性变化的方法。

对象的优点：

  • 模块化
  • 隐藏内部实现细节
  • 代码重用
  • 插件化、易于调试

# 类 class

通过构造类得到实例对象。类就像模板一样。

# 继承

子类通过继承得到父类中通用的属性和方法。

# 接口 interface

接口定义了方法的输入和输出，继承了接口的类必需按照这个标准实现方法。

# package

等于命名空间，方便组织代码

# 语法基础

# 变量

  • Instance variables (Non-static Fields): 属于实例本身，独立于其他个体的同名变量
  • Class variables (Static Fields): 属于类，独一份，不管实例数量有多少（亦可使用 final 关键字指示其不可变性）
  • Local variables
  • Parameters

# 命名

  • 大小写敏感，以 $ 、 _ 或者字母起头，但惯例一般不推荐以符号作为命名起始
  • 驼峰式命名；常量全大写，以下划线连接，此情况以外一般不再使用下划线

# 类和对象

# 类 class

一个子类 subclass 只能继承自一个父类 superclass ，但可以实现多个接口 interface

  • 类中的成员变量：域（ fields ）
  • 方法或代码块中的：local variables
  • 方法声明中的：parameters

权限修饰符

  • public
  • private

# overloading

类内部的方法可以重名，java 可以自动分辨不同入参类型的同名方法

编译器无法区分返回类型，所以不要提供多个相同签名（命名 + 入参列表）但返回不同的方法

尽管如此，还是应当尽可能少用同名方法的重载，这会降低代码的可读性

# 权限修饰符

修饰符 Class Package Subclass World
public Y Y Y Y
protected Y Y Y N
no modifier Y Y N N
private Y N N N

# Nested Classes

非 static 的嵌套类，可以访问它所在的类的其他成员，static 修饰的则不可以

嵌套类可用的修饰符

  • private
  • public
  • protected
  • package private

为什么使用嵌套类？

  • 在逻辑上组织仅在此处使用的 classes
  • 提升封装性
  • 易读易维护

因为 JVM 可能运行在不同环境上，创建的 synthetic constructs 会有差别，所以序列化对象后在反序列化可能导致不同的结果。

# 匿名类

HelloWorld frenchGreeting = new HelloWord() {
    String name = "tout le monde";
    public void greet() {
        greetSomeone("tout le monde");
    }
    public void greetSomeone(String someone) {
        name = someone
        System.out.println("Salut " + name);
    }
}

构成：

  • new 操作符
  • 需要被实现的接口或被扩展的类
  • 括号内传给构造器的参数
  • 类的声明体

匿名类没有 construtor

# Lambda 表达式

为了清减和易读，它允许你把方法 / 代码当做数据作为参数传递。

(int x, int y) -> {
    Log.i("TAG", "x:" + x + " y:" + y);
    return x + y;
}

构成：

  • (int x, int y) ：参数部分
  • -> ：箭头 goes to，指向代码块
  • 用 {} 包裹的代码块

# 方法引用

如果一个方法没有任何返回只是执行了已有方法的话，使用 lambda 表达式就会显得略微复杂些了

Array.sort(rosterArray, Person::compareByAge)

# interface & inheritance

# interface

只能包含：

  • constants
  • method signatures
  • default methods
  • static methods
  • nested types

方法体只能在 default methods 和 static methods 中存在

interface 不可以被实例化，它只能被 implement 或者 extend

接口间的继承，也遵循就近原色，如果方法签名或方法本身被重载了，使用的最近的定义。

# overrides

定义与父类中方法有相同签名的方法：

Superclass Instance Method Superclass Static Method
Subclass Instance Method Overrides Generates a compile-time error
Subclass Static Method Generates a compile-time error Hides

# Polymorphism 多态

super.functionName()

# 关键字 super

  • 获取父级类的成员
  • 获取父类的构造器

# final classes and methods

可以用 final 声明类的方法，该方法不可改变。在构造器中调用的方法应当被声明为 final ，以避免因重载造成意料之外的问题。

用 final 声明的类，不可被继承。

# abstract methods and class

都不可以被实例化。

区别：抽象类可以声明非 static 和非 final 的域，可以定义 public 、 protected 、 private 的具体方法

# 泛型

暂时不指定类型，等到使用时再决定。

泛型增加了代码的稳定性，并在编译阶段使更多问题得以检测。

List<String> list = new ArrayList<String>();
list.add("Hello");
String s = list.get(0);
  • 使用泛型类时，创建对象的时候必须指定类型参数的具体类型。
  • 使用泛型方法时，不必指明参数类型，编译器会自行推断出所需的具体类型。
