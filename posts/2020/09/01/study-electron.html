# 主进程和渲染进程

Electron 运行 package.json 的 main 脚本的进程被称为主进程。一个 Electron 应用总是有且只有一个主进程。主进程使用 BrowserWindow 实例创建页面。每个 BrowserWindow 实例都在自己的渲染进程里运行页面。

如果想在 web 页面里使用 GUI 操作，其对应的渲染进程必须与主进程进行通讯，请求主进程进行相关的 GUI 操作。

TIP

不要直接使用 nodeIntegration: true ，而是要正确使用 preload，这一点极为重要！！！ 指路 stackoverflow：How to use preload.js properly in Electron

截止目前通信遇到的问题大多都是没有关联到正确的 view。

另外，主进程或渲染进程不能直接获取到页面内容的信息， executeJavascript 这个 API 也不能直接拷贝 window``navigator 等复杂对象作为结果，最好字符串化， executeJavascript 同样也不能接受使用 async/await 关键字产生的返回结果为 promise 的方法，尤其需要注意。

# 为什么我在 render 进程取不到 remote？

electron-remote-is-undefined

需要在对应的 BrowserWindow 或 BrowserView 的 webPreferences 中设置 enableRemoteModule 为 true ，默认情况下也是开启的

但是 remote 是为了在渲染进程中能够使用主进程模块，这个似乎不太符合主进程渲染进程分离的原则，尽量还是用进程间通信？
