学习笔记 之 队列、栈

# 前置说明

# 为什么需要

数组可以通过索引访问 随机 元素。但是某些情况下，我们可能需要限制 处理顺序。

处理顺序分两种，并对应两种数据结构：

  • 先入先出（First In First Out）：队列
  • 后入先出（Last In First Out）：栈

# 能解决什么样的问题

  • 广度优先搜索（Breadth First Search）
  • 深度优先搜索（Depth First Search）以及深入理解递归

# 队列

队列是线性数据结构，涉及操作：入队、出队。

但是某些情况下可能存在效率很低、浪费空间的问题，此时考虑 循环队列，目的是重用以减少浪费。

用 (tail + 1) % length == head 判断循环队列是否已满。

# 题后思考

注意题目条件，先理清要从哪里开始，遍历到节点时的处理是什么。

像墙与门是先确定门节点，然后从门节点开始遍历，并记录步数。

像岛屿数量则是找到未被记录过的土地节点开始遍历相邻土地，遍历时要注意避免重复计数。

# 栈

操作：入栈、出栈、获取栈顶元素

最小栈问题：维护两个栈，一个栈负责正常的数据出入，另一个栈维护由底向上降序的数据，后者栈顶的元素一定是前者中最小的值

# 题后思考

考虑是否需要按照某种顺序维护数据，或者数据内容是否有嵌套、匹配一类的条件。

在深度优先遍历顺序中，找到的第一条路径可能不是最短路径。

# 小结

记住队列和栈的特点（先进先出与后进先出），考虑问题是否有相同的特征再去应用。

队列的扩展问题：双端队列、优先问题。
