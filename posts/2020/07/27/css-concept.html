# block , inline , inline-block

  • 块级元素在水平方向上延展至占据一整个行，可以设置竖直方向的边缘空白（vertical margin）
  • 行内元素在水平方向上延展至足够包含内容，不可以设置竖直方向的边缘空白
  • 行内块级元素与行内元素很像，但是可以设置竖直方向边缘空白

# width 的计算

盒模型包括三个部分

  • content area
  • padding area
  • border area

宽度的计算取决于 box-sizing 的值，默认值 content-box 表示 width 适用于内容盒，整个盒模型的宽度则是 width + 2 * padding + 2 * border ， border-box 表示 width 适用于包含 border 部分的盒，整个盒模型的宽度即为 width ，而此时内容盒的宽度则是 width - 2 * padding - 2 * border

此处不讨论 margin 因为它表示两个元素的间距，并不在计算范围之内

# margin 的折叠

父元素设置 display: flex 和 display: grid 时，子元素的边距不会折叠

# 相对单位的使用

以像素 px 作为单位很直白清晰，但在内容的缩放或重置尺寸时体验可能不好

理解 em 与 rem 的区别：

  • em : em 值的大小是动态的。当定义或继承 font-size 属性时，1em 等于该元素的字体大小。如果你在网页中任何地方都没有设置文字大小的话，那它将等于浏览器默认文字大小，通常是 16px。
  • rem : root em，始终相对根元素计算。

# 基线 baseline

书写英语字母时，字母 x 底部所在的位置

在 inline-block 元素中，它的基线是正常流中最后一个行盒（line box）的基线，但是，如果这个 line box 里面没有 inline boxes 或者其 overflow 属性值不是 visible，那么其基线就是 margin bottom 的边缘。

# 行高 line-height

两行文字基线之间的距离

# 单位 ex

指的是小写字母 x 的高度，即 x-height

# vertical-align

vertical-align : 指定行内元素或者表格单元格的垂直对齐方式。

一旦元素被设置为浮动定位 float 或绝对定位 postion: absolute; ，该元素即强制被当做块级元素处理， vertical-align 将失去作用。

vertical-align: middle; 这个规则指的是，由 baseline 向上 1/2 个 x-height 的高度，并非绝对的垂直居中对齐。

# 幽灵空白节点导致的额外间距

解决办法：

  • 将内容元素设置为 display:block（利用 vertical-align 的生效前提）
  • 将 vertical-align 设置为 top，bottom，或者 middle 等值（利用属性值的表现行为）
  • 将 line-height 设置为 0（利用 line-height 为 0 时，基线上移）
  • 将 font-size 设置为 0（如果 line-height 的值为相对值）
  • 设置浮动或者绝对定位（如果布局允许的话）

# BFC

BFC (Block formatting context) 直译为 "块级格式化上下文"。它是一个独立的渲染区域，只有 Block-level box 参与，它规定了内部的 Block-level Box 如何布局，并且与这个区域外部毫不相干。

我们常说的文档流其实分为定位流、浮动流和普通流三种。而普通流其实就是指 BFC 中的 FC。

FC 是 formatting context 的首字母缩写，直译过来是格式化上下文，它是页面中的一块渲染区域，有一套渲染规则，决定了其子元素如何布局，以及和其他元素之间的关系和作用。 常见的 FC 有 BFC、IFC（行级格式化上下文），还有 GFC（网格布局格式化上下文）和 FFC（自适应格式化上下文），这里就不再展开了。
