# 请求的 data 构造

除了 param 是通过上一步获取到的数据以外，其他都是写死的数据

{
    "songinfo": {
        "module": "music.pf_song_detail_svr",
        "method": "get_song_detail_yqq",
        "param": {
            "song_type": 0,
            "song_mid": "002ehbCz4LvX1t",
            "song_id": 270006366
        }
    }
}

经过尝试， param 中仅携带 song_id 即可获得需求的数据

# 内容完整性校验

通过挂载在全局的 getSecuritySign 方法，将 data 字符串化作为入参生成校验标志

很疑惑，不晓得为什么这个方法在 node 环境的结果可以被校验通过，但是在浏览器环境下就没有通过，待调查

# 构造播放 url

相关代码在播放器页的 player_module_[hash].js 文件中

可播放路径格式： CDN + VKey

CDN 有两种：

  • http://isure.stream.qqmusic.qq.com
  • http://ws.stream.qqmusic.qq.com

VKey 仍是通过接口请求获取的，包含歌曲信息： /C400002ehbCz4LvX1t.m4a?guid=4379097852&vkey=DAEFFCCFBAB8C78A8167DF13158CCA1A737A63071680DE4DC620A94C0E146B90EB6DF99D966AA4506CFD2A6F2C8C6B86EE40DE8A92B2520C

生成 CDN 地址后面的具体歌曲播放信息的关键方法： getVKey

请求的 url 依然是 //u.y.qq.com/cgi-bin/musicu.fcg ，请求 data 格式如下：

{
    req_0: {
        module: "vkey.GetVkeyServer",
        method: "CgiGetVkey",
        param: {
            guid: "" + _getGuid(),
            songmid: "002ehbCz4LvX1t",
            songtype: 0,
            uin: "",
            loginflag: 0,
            platform: "20"
        }
    }
}

其中关键的方法： _getGuid 方法只是获取某种随机化的时间戳

function _getGuid() {
    var t = (new Date).getUTCMilliseconds();
    return _guid = Math.round(2147483647 * Math.random()) * t % 1e10
}
