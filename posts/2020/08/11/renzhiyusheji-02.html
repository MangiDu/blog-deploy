# 第四章：色觉是有限的

# 色觉的原理

眼睛的视网膜有两类感光细胞：

  • 视杆细胞
    • 可以察觉光线强度但感觉不到颜色
  • 视锥细胞
    • 可以察觉颜色，分为三类，分别对红色、绿色、蓝色光敏感

但是视杆细胞只在低亮度时工作，在现代社会白天和人工照明的情况下，视杆细胞则完全过曝了。大部分时间里我们的视觉完全依赖于视锥细胞。

然而每种视锥细胞并不是单纯的分别对红、绿、蓝色光敏感。每种视锥细胞敏感的光谱范围都比你想的要广，而且三者互相重叠，但敏感程度相差很大。

而大脑对于综合各种视觉信息的处理办法是做减法（颜色对抗通道：红 —— 绿，黄 —— 蓝，黑 —— 白）。

导致的结果就是我们的视觉系统对颜色和亮度的差别，即对比和反差鲜明的边缘，比对绝对的亮度要敏感。

# 颜色的呈现方式影响视觉的区分能力

  • A. 两个颜色越浅（不饱和），越难区分
  • B. 两块颜色越小，越难区分
  • C. 两块颜色相距越远，越难区分

# 使用颜色的准则

准则 补充说明
用饱和度、亮度、色相来区分颜色 确保颜色之间有较高的反差，如果在灰度模式下不能区分，那么颜色间的差别就不够
使用独特的颜色 在一个颜色对抗通道触发强信号，在另外两个通道上空信号的颜色是最容易区分的，即：红、绿、黄、蓝、黑、白色
避免使用色盲的人无法区分的颜色对
在颜色以外提供其他提示 如 icon 等
将强烈的对抗色分开 强烈的对抗色放在一起会让人有难受的闪烁的感觉

# 第五章：我们的边界视觉很糟糕

  • 处于人视野边界的暗色静止物体经常不会被注意到
  • 边界视野中物体的运动通常会被察觉

由于视锥细胞的分布、视神经连接比例、大脑视觉皮层的资源分配不均匀，导致人类视野中央的视觉分辨率要远远高于其他地方。

我们视野的中央凹大约相当于一臂距离外一片拇指指甲的大小。

边界视觉分辨率非常差，但却没有造成 “失去部分视觉” 的感觉，是因为眼球大约以每秒三次的速度不断快速移动，选择性地将焦点投射在周围的环境物体上。大脑则以粗犷印象派的方式，基于我们的认知与期待，填充视野的其他部分。

但是边界视觉也有它的重要作用：

  • 引导中央凹：提供低分辨率的线索，帮助大脑计划往哪里以及什么顺序移动眼睛
  • 察觉运动：人类祖先为了更好地生存下来，需要快速发现食物和躲避捕食者的能力，这使得我们视觉边界的任何轻微运动，都很可能引发注意，但大脑的控制是潜意识、自动的、非常快速的
  • 让我们在黑暗中看得更清楚：正如视杆细胞在光线良好的情况下会超负荷，低亮度下视锥细胞无法很好工作，所以视杆细胞接管了这个功能，此时，视杆细胞主导的视觉成为暗视觉。（因为中央凹没有视杆细胞，在低亮度条件下，如果不直接看物体，反而能看的更清楚）

# 让信息可见的常用办法

  • 放在用户所看的位置上：注意力可预期的位置
  • 标记出错误，使用错误符号：明显而靠近的位置
  • 保留红色以呈现错误信息：认知习惯

# 更强但慎用的办法

  • 弹出式对话框
  • 使用声音
  • 闪烁或短暂的晃动

# 视觉搜索

如何更好地让边界视觉引导中央凹：让目标 “跳入” 边界视觉内。（如加粗的字体、一片黑色中的红色字体）

面多多个物体的搜索时，可以借助符合通识认知习惯的图案或图标。

# 第六章：阅读不是自然的

说话和理解语言是自然的人类活动，阅读不是。

在更久远的数万数十万年前，人类就逐步进化出了口头语言需要的神经结构，但阅读和写作近几千年才出现，实际上这是一种人造的、通过系统的指导和训练才获得的能力。

一个人的阅读能力与特定的语言和书写系统有关。

# 学习阅读 = 训练视觉系统

但人眼只有中央凹和近中央凹的很小一部分被训练出了识别阅读过程中文字模式的能力。这个过程同样需要大脑系统的协调（控制眼睛的运动，以特定方式浏览文字）。

我们以为阅读是平滑平稳的过程，实际上不是，视线是在不断跳跃的，它通常遵循文字布局，短暂地停留几分之一秒获取基本文字样式，然后跳跃到下一个重要的词汇，常见的功能词连词等是被近中央凹探测或者干脆被大脑推断出来的。

中间对齐的文字会干扰大脑对于下一行起始位置的猜测。

正因为我们经过训练才会阅读，所以阅读时眼球注视点对周围信息的感知是不对称的。

# 阅读的模式

  • 自下而上，特征驱动
    • 从线、边、角等（形状的）基本特征到词素、单词、短语进行非自动的、有意识的识别和分析，经过足够的训练，这可能成为无意识的过程
  • 自上而下，语境驱动
    • 从完整句子、段落主旨到单词和字符，从知道文字的含义开始，利用事先的了解去弄清楚或者猜测这个高层模式各个组成部分应该是什么。（大脑会对还没读到的文字进行预测，从而进行略读）这不太可能成为无意识的，因为大部分语言模式和语境不会频繁到能够形成特定的神经触发模式。

研究证明：阅读的主要方式是无语境的、自下而上、特征驱动的方式，这需要掌握到无意识的程度。而语境阅读是一种候补的方法，在特征驱动阅读存在困难或者不能做到无意识时才起作用。

对不熟练的阅读者，基于特征的阅读不是无意识的，而是有意识的费劲的，一部分注意力被分放在识别一串串单词上，使得他们没有更多精力构建语句和段落的含义，导致对内容缺少理解。

医学技术则从某种角度证实了这一点：扫描证实初学程度和熟练程度的阅读者在阅读时，大脑的神经通路和激活区域是不同的。

# 信息的设计对阅读的影响

阻碍阅读：

  • 不常见、不熟悉的词汇，举例如：“您的会话已过期，请重新认证” 和 “您的会话已过期，请重新登录”，对非技术人员来说 “认证” 并不是能马上理解的词语。
  • 难以辨认的书写和字形，举例如：全大写的英文文章和正常的；印刷体和手写草书。
  • 微小的字体难以识别（书中没有申明理由，个人猜测还有一部分原因是由于字体过小导致按照以往阅读习惯单次眼跳时中央凹需要处理的信息变多了造成了负担）
  • 嘈杂背景下的文字，举例如：区分真人与爬虫的噪点验证码图片
  • 信息中多是重复的内容，会让人忽略其中真正核心的差别
  • 居中对齐的文字，这与我们被训练的每行回到同样水平位置的无意识阅读是有冲突的，会减慢阅读速度

因此，对于设计者来说，设计的目标应当是支持阅读，而不是干扰这个过程。

  • 避免之前提到的缺陷
  • 使用有限的、高度一致的词汇
  • 将文字格式设计出视觉层次，以便浏览轻松

另外，如果可能，尽量减少阅读的需要。在 GUI 中尽可能减少文字量，在用户手册中，使用最少的文字让用户完成目标。在产品描述中，提供简要的综述，在用户提出具体需求时再提供详细内容。

# 第七章：我们的注意力有限，记忆力也不完美

记忆分短期记忆与长期记忆。

过往的认知认为短期记忆像是内存，长期记忆像是硬盘，是两套存放位置不同的存储。但近年的医学研究表明，这二者是一套记忆系统，并与感知有着紧密的关系。

# 长期记忆

记忆的形成：由参与某神经活动模式的神经元上长期甚至永久的变化组成，这使得该模式很容易被再次激活。其中某些变化将某种化学物质释放到神经元周围区域，以此改变了神经元对刺激的敏感度，直到这些物质被稀释或中和。而神经元的生长和神经元之间的新连接的形成则造成更永久的变化。

激活记忆就是再次激活与记忆产生时同样的神经活动模式。一个神经记忆的模式越经常被激活就越容易被激活。

记忆不是被锁定在大脑某个特定的地方，而是涉及了一个延伸到很大区域的神经网络。不同记忆的神经活动模式因共享的感觉特征而相互覆盖。移除、破坏或者抑制大脑某个部分的神经细胞并不能完全清除这些记忆，而仅仅是降低了记忆的细节与精确程度。但是一些参与神经模式的区域可能处于关键路径上，移除、破坏或者抑制他们会导致模式不再被激活，也因此完全消除了其对应的记忆。

# 短期记忆

实际上是感觉、注意以及长期记忆留存现象的组合。

在人类大脑的注意机制中，意识专注于感觉和被激活的长期记忆中非常小的子集（构成短期记忆的主要部分）而忽略其他部分，这种活动被称为工作记忆，我们的讨论也仅限定在工作记忆上。

工作记忆并不是存储区域，它是在短期内感觉和长期记忆中被激活我们可以意识到的部分。它是若干注意焦点的组合，能力有限并且内容不稳定。

# 注意力与工作记忆

专注的时候，你意识不到周围大部分事物的动向。感知系统与大脑对周围事物的采样选择性极高，因为它们没有能力处理所有的信息。

第 1 章描述了感知基于目标的过滤和挑选作用，但除了关注的目标有关的物体和事件之外，注意力还通常会被以下目标吸引：

  • 移动，特别是朝我们移动的物体
  • 威胁
  • 人脸
  • 性和食物

# 注意力的容量就是工作记忆的容量

工作记忆的特征：较低的记忆容量和易失性。

那我们的记忆仓库中究竟有多少探照灯呢？四个。某些实验研究表明，当被展示的对象互相之间无关联时，工作记忆的容量在三到五个左右。

工作记忆的不稳定性在于注意力转移到新事物上就得从之前关注的事物上移开。

工作记忆的丢失不仅适用于物品细节的记忆，也适用于对我们的目标的记忆（一下子忘记要做什么或者某件事做到哪一步了）。

# 工作记忆对 GUI 设计的影响

最基本的启示：帮助用户从某一时刻到下一时刻记住核心的信息，不要要求用户记住系统的状态或者他们已经做了什么，因为他们的注意力专注于主要目标和朝向目标的进度。

# 模式

很多交互系统提供模式，使同样的控件在不同模式下提供不同的功能。模式让交互系统分配不同的意义给同样的操作从而减少用户必须学习的操作的数量。

但人们常常忘记自己处在哪种模式中而引发错误的操作，因此很多设计准则提到 “要么避免模式，要么提供强烈反馈告知用户当前的模式”。

# 搜索结果

当用户得到搜索结果并开始关注时，他们常常忘记自己之前的搜索词是什么，这是因为注意力已经转移的缘故。在搜索框中提供搜索词，以减少工作记忆的压力。

# 行为召唤

不要在同一个页面内放置多个相互竞争、夺取用户注意力的 “行动召唤” 元素。每个页面应当只放置一个占主导地位的行动召唤元素，这样才不至于超出用户的注意能力，甚至于引导用户无法完成目标。

可以利用 “流程漏斗（process funnel）” 的设计准则引导用户完成目标。

课外阅读：Over-controlling a Process: The Funnel Experiment

# 指令

在多步操作中用来展示说明的交互系统，应当允许人们在完成所有操作步骤的过程中随时查阅使用说明。

举例：如地图 app 中换乘路径的截图保存。

# 导航深度

如何引导用户到他们需要的信息或目标处，对于大部分用户而言，宽而浅的导航层级结构比窄而深的结构更易于使用。这一理论完全适用于应用程序窗口、对话框以及菜单层级结构的设计。

而在超过两个层级的结构中，面包屑式的导航路径能够提醒用户他当前正处于什么位置。

# 长期记忆的缺点

  • 容易出错：大脑使用高压缩比的方法导致大量信息的丢失。图像、概念、事件、感觉和动作都被减弱到抽象特征的组合，不同记忆则按照特征的多少去记录。
  • 印象派
  • 异质
  • 可回溯修改
  • 也容易被记忆或获取时的很多因素影响
    • 受情绪影响

# 长期记忆对 GUI 设计的影响

启示在于，人们需要工具来加强长期记忆。从结绳记事到如今的云笔记，都是人类发明的帮助自己长期记忆事物的技术。

因此，软件设计者应当避免给长期记忆造成的负担。

举反例如：

  • 用户修改密码，系统要求数字容易记忆，却又要避免一系列常见组合，又要求用户自行写下密码
  • 修改密码的安全问题，“你养的狗的名字”，如果用户养了多条狗呢？

以上是内容设计，而对于交互来说，界面的一致性是有助于学习和长期保留的。虽然一致性被一些人批判为定义不清和容易错误应用设计准则。
