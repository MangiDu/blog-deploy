# 设计准则从何而来

基于人类心理学：人类如何感知、学习、推理、记忆，以及把意图转化为行动。

# 第一章：我们的感知存在偏差

（过去的）经验、（现在的）环境、（将来的）目标都会影响我们对世界的真实感知，设计时应当把这些影响因素考虑在内。

# 经验

  • 先入为主的经验会影响感知
    • 大部分人对这个图第一反应是随手泼洒毫无规律的墨点。
    • 但如果被告知这是一只在嗅树附近地面的斑点狗之后，就很难再把这张图看成随机无序的点了。
  • 感知的模式或框架
    • 我们生活中大部分时间都在熟悉的环境中度过，不断置身各种环境在我们的心智中建立起模式，让我们对不同的地方有不同的期待。这种感知模式被称作框架，包括在各个环境中通常遇到的事件和对象。
    • 框架影响下的感知，不必不断检查身边环境的细节，这是心智应付世界的捷径，但也会让人看到或感受到其实不存在其实未完成的东西。
  • 习惯性
    • 重复置身同样或类型的环境中会降低感觉系统的敏感性。
    • 当 “你是否确定” 的对话框一次又一次弹出，最开始你会注意到并且或许做出反应，但最终你会反射般直接忽视并关闭对话框。
  • 注意瞬脱
    • 过往经验对底层感知的另一个影响，发生在人们刚刚发现或者听到某件重要的事情之后。在识别之后的 0.15~0.45 秒之间，即使耳朵和眼睛正常工作，人们也接近于失聪而且无视其他视觉刺激。这个现象被称为注意瞬脱，研究者们认为这是由于大脑的感觉和注意力在短时间内完全用于处理第一个识别而产生的。
    • 当人们在使用基于电脑的系统或在线的服务时，如果事情连续发生的太快，他们会因为注意瞬脱而错过一些信息或者事件。

# 环境

一般可能认为视觉感知是自下而上工作的，以阅读为例，即首先识别字母，再组合为单词，然后形成句子。但实际上，阅读的视觉感知，并不完全是自下而上的，其中也有自上而下的作用，比如包含某字母的单词能够影响我们的判断。

类似的，对文本语义的理解能影响我们看到的单词，而上下文语境也能使同一个单词出现不同的含义。

视觉感知中不止阅读会因环境影响造成偏差，不同感官之间也会有感知偏差，甚至五官之间会相互影响。

识别一个字母、一个单词、一张脸或其他任何物体的神经活动，都包含了环境刺激产生的神经信号的输入。这个环境包括感知到的其他临近对象和事件，甚至由环境激活的、对以往感知到的对象和事件的记忆。

# 目标

“鸡尾酒会” 效应：如果你在一个拥挤的酒会上与人谈话，你能把大部分注意力集中在他的话上，即使身边还有很多人在谈话。你对谈话的兴趣越大就越能过滤掉周围的对话，如果你对谈话内容感到乏味了，多半就会越来越多地听到周围的谈话。

目标过滤的效应在成年人身上特别可靠，成人比儿童对目标更专注。儿童更容易被刺激驱使，目标较少地过滤他们的认知。这个特点使他们比成人更容易分心，但也使他们观察时更不然容易产生偏差。

目标影响感知的机理是什么？

  • 影响我们注意什么：感知是主动的，也不是对周围事物的简单过滤，是对世界的体验和需要理解的东西的获取，那些能够引导完成目标的对象就会吸引我们，而与目标无关的则会或多或少被忽略。
  • 使我们的感知系统对某些特性敏感：在寻找某件物品时，大脑能预先启动感官，使他们对要寻找的东西变得非常敏感。

# 设计中的考量

  • 避免歧义
    • 确认所有用户对信息的理解是一致的。当无法消除歧义时，要么依赖标准或管理，要么告知用户用你期望的方式去理解歧义。
  • 保持一致
    • 在一致的位置摆放信息和控件。不同页面上提供的相同功能的控件和数据的显示应该摆放在每一页上相同的位置，而且他们应当有相同的字体、颜色、阴影等。
  • 理解目标
    • 设计者应当了解用户使用系统的目标，要认识到不同用户的目标可能不同，而且他们的目标强烈左右着他们能感知到什么。在一次交互的每一个点上，确保提供了用户需要的信息，使他们能够注意到并使用这些信息。

# 第二章：我们的视觉经过优化更容易看到结构

# 格式塔原理

视觉感知的格式塔原理：人类的视觉是整体的，我们的视觉系统自动对视觉输入构建结构，并且在神经系统层面上感知形状、图形和物体，而不是只看到互不相连的边、线和区域。

视觉感知有一些倾向和针对性：

  • 试图给对象分组
    • 接近性
    • 相似性
  • 试图解析模糊或者填补遗漏来感知物体
    • 连续性
    • 封闭性
  • 分解复杂场景来降低复杂度
    • 对称性
  • 组织数据
    • 主体 / 背景
  • 运动的物体
    • 共同命运

在设计之后，可以使用这些格式塔原理来考量各个元素间的关系是否符合设计的初衷。

# 接近性

物体之间的相对距离会影响我们感知它们是否以及如何组织在一起。互相靠近（相对于其他物体）的物体看起来属于一组，而那些距离较远的就不是。

根据接近性原理，可以通过拉进某些对象之间的距离，拉开与其他对象的距离使它们在视觉上成为一组，而不需要分组框或者可见的边界。许多图形专家推荐这一方式来减少用户界面上的视觉凌乱感和代码数量。

# 相似性

如果其他因素相同，那么相似的物体看起来归属于一组。

# 连续性

我们的视觉倾向于感知连续的形式而不是离散的碎片。

# 对称性

我们的视觉区域中的信息有不止一个可能的解析，但我们的视觉会自动组织并解析数据，从而简化这些数据并赋予它们对称性。

一对叠加的菱形比其他解释更简单，边更少并且更对称。

利用视觉系统对对称性原理的依赖，可以用平面显示来表示三维物体。

# 主体 / 背景

我们的大脑将视觉区域分为主体和背景。主体包括场景中一个占据我们主要注意力的所有元素，其余的则是背景。

我们对主体和背景的差别感知受到场景特点和观察的焦点决定。

# 共同命运

共同命运涉及运动的物体，共同命运原理与接近性和相似性原理相关，都影响我们所感知的物体是否成组。

该原理指出，一起运动的物体被感知为属于一组或者是彼此相关的。

共同命运暗示共同历程，在一些动态模拟中可用以展示不同实体的关系。

# 第三章：我们探索和利用视觉结构

当信息以简洁和结构化的方式呈现时，人们更容易浏览和理解。

但除了把信息变得精炼、结构化和不重复还不够，还必须遵从一些图形设计的规则（2 章已经介绍了一些），使用户不必依靠主动思考就能明白信息之间的关系。

如 UI 库中常用到的日期组件，组合输入框等。

# 视觉层次让人专注于相关的信息

可视化信息的重要目标之一：提供一个视觉层次。

其信息的布置安排能够：

  • 将信息分段，把大块的信息分割为各个小段
  • 显著标标记每个信息段和子段，以便清晰地确认各自的内容
  • 以一个层次结构来展示各段及其子段，使得上层的段能够比下层得到更重点的展示
