# 目的

业务上：“职权” 下放，降低无开发团队进入门槛，提升效率和体验

打通上下游生产资料壁垒

低代码兼具灵活与效率，但也需要平衡好代码开发与可视化开发间的关系

但是复杂度是固定的，它可以转移但不可能凭空消失，既然可视化开发不能要求使用者拥有开发者一样的技能素质，其背后的工作就要仔细评估

# 怎么做

代码用于生产零部件，可视化用于组装。形成上下游分工的产业链式结构。

# 组件的拖放

单纯用 drag API 不够用了，而且也不合适，拖放的 shadow 样式和 html 元素不可控是重要原因之一

所以还是用 mousedown、mousemove、mouseup 来模拟这个过程

# 图层层级

如果只考虑简单情况，那么每个组件都是单独的一个图层

但是这也并不是理想情况，因为一些组件间天然有包含关系

利用数组的序列关系处理图层对象，图层由低到高，例如：置顶操作就是把图层对象放到数组队列最后一个

# 撤销、重做

记录几次操作得到的组件树结果就可以了，回退就是切换指向这个数组的 index

正常操作下，index 随着 history 数组而增加，回溯时改变 index 就好了

但要注意数组的拷贝，一定是深拷贝
